Changelog

v1.25
Baby tab: 24h summary now shows mL totals for Breast and Formula instead of entry counts (an entry only counts toward one or the other if exactly one is checked — ambiguous entries are excluded from both). Monitor tab: added a connected-viewer list with per-viewer disconnect, a live "talking" indicator tied to each viewer's Push to Talk state, and a sound-activated alert — the monitor analyzes its own mic independently of the video call and notifies viewers (vibration, chime, and a system notification if permitted) when sound crosses a threshold for a sustained moment, with a cooldown so continuous crying doesn't spam repeated alerts.
v1.24
Added Monitor tab — baby-monitor mode, distinct from Stream's two-way call. Anyone opening the tab can start monitoring (broadcasts camera+mic) or, if a monitor is already active, view it. Only one monitor at a time — starting a new one displaces the old. Video flows monitor→viewer only; audio is two-way but only between monitor and each viewer individually (never viewer-to-viewer). Monitor mic defaults on with its own mute toggle; viewer mic defaults off behind a tap-to-toggle Push to Talk button. Includes a disconnect watchdog with an audible chime and a hard-to-miss alert (faster than Stream's reconnect threshold, since a viewer device may be unattended on a nightstand), a night-mode screen dim that keeps audio/connection alive, and a live viewer count shown on the monitor's screen.
v1.23
Hid the Stream and Media tabs from the tab bar to make room for the upcoming Monitor tab. Code for both is untouched and still fully present in the file — panels, JS, CSS — just not reachable from the UI for this build. Easy to re-enable later.
v1.22
Fixed Stream tab not connecting at all — calling code referenced initSupabaseClient(), which had been left undefined during the v1.21 Stream tab rebuild, so the signalling channel never subscribed and both sides waited forever.
v1.21
Stream tab rebuilt to share code with IPM's CommBadge: same card layout, collapsible Settings panel, camera toggle button (replacing Rear/Front selector), and call logic. Added Wake Lock (keeps screen on during calls), facingMode fallback if the requested camera is unavailable, and automatic video/reconnect recovery when returning from background — none of these had actually made it into Matrix despite being listed as shared fixes. Tap-to-hear-audio prompt added for when iOS blocks remote audio autoplay. Baby tab: moved Pumped under Other (now first), added a new Type row with Breast and Formula buttons, tracked in the log going forward.
v1.20
Fixed manifest start_url, scope, and app name (was still showing Baby Log) — fixes home screen shortcut 404. Added audio routing fix and stricter audio constraints to Stream tab. Added camera Flip button for mid-call switching, synced from CommBadge.
v1.19
Rebuilt Stream tab as two-way video call with session chat. Both users send and receive video and audio. Single Start Call button — second user sees Join Call. Busy signal for third users. Data channel chat appears during call, cleared on end. Picture-in-picture local video. End Call button.
v1.18
Added stream quality sliders (resolution, frame rate, bitrate) shown before going live, saved to localStorage. Default resolution raised to 480×360. Bitrate, framerate and SDP cap all driven by slider values. window._debugPC exposed for console diagnostics.
v1.17
Added TURN server (Open Relay) for NAT traversal. Adaptive bitrate adjusts based on packet loss and RTT. Forced keyframe every 2 seconds for faster recovery. ICE restart on disconnection before full teardown. Auto-reconnect viewer up to 5 attempts with backoff. Connection quality indicator (Good/Fair/Poor) in stream status bar.
v1.16
Rebuilt stream signalling as pull-based: viewer announces ready, broadcaster responds with offer. Broadcaster stays live indefinitely, viewers join any time. Opening Stream tab auto-joins if stream active. Bitrate back to 200kbps for stability. Removed order dependency entirely.
v1.15
Raised stream bitrate to 400kbps. Set degradationPreference to maintain-framerate for smoother motion. Added Mute and Hide Camera buttons for broadcaster. Check for Update now returns to Settings tab after reload.
v1.14.4
Stream presence via Supabase stream_session table. Broadcaster can go live before viewer joins. Viewer checks if stream is active on Watch tap and connects immediately. Stream tab shows live indicator to all users. Retry logic if first request-offer is missed.
v1.14.3
Lowered stream quality to reduce bandwidth: 320x240 resolution, 12-15fps, 200kbps bitrate cap via SDP and RTCRtpSender. VP8 codec preferred. Lower audio sample rate. Should prevent freezing on modest connections.
v1.14.2
Fixed frozen video: ICE candidates now queued until remote description is set. Duplicate offers ignored mid-negotiation. Explicit play() call added for iOS Safari. Broadcaster resets cleanly on viewer disconnect.
v1.14.1
Fixed stream: viewer no longer requests camera. Broadcaster re-sends offer every 4 seconds until viewer connects. Viewer requests fresh offer on join. Broadcaster auto-resets for reconnection.
v1.14
Added Stream tab with live WebRTC video and audio streaming via Supabase Realtime signalling. Either device can broadcast or watch. Camera selector (front/rear). Live indicator on Stream tab. Code cleanup: storageFetch helper, separated load/render functions.
v1.13
Renamed app to The MurDough Matrix. Tabs moved to top. Removed 24H tab — totals now at bottom of Baby tab. Live status moved to Settings. Note bar rebuilt with Edit button.
